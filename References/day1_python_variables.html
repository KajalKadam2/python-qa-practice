Day 1 Python lesson — variables, strings, integers, booleans with 5 QA automation examples

What is a variable?
A variable is a named box that holds a value. You give the box a name, put something in it, and refer to it by name later.

In Python you don't declare a type — you just write name = value and Python figures out the type automatically.
The 4 types you learn today
str String
"any text in quotes"
'single quotes work too'
int Integer
5000   200   0   -1
whole numbers, no quotes
bool Boolean
True    False
capital T and F always
float Decimal
3.14   0.5   99.9
numbers with decimal point
Naming rules — important
Variable names: lowercase, words separated by underscores.
Good: base_url   timeout_ms   is_logged_in
Bad: BaseUrl   timeout ms   2url (can't start with number)

Example 1 — Test environment config str + int + bool
Every automation project has a config — the URL to test against, the browser, the timeout. This is the first thing you store in variables.
Type this into day1.py and run it
# Test environment configuration base_url = "https://staging.myapp.com" browser = "chromium" timeout = 5000 headless = True retry_max = 3 print(base_url) print(browser) print(timeout) print(headless) print(retry_max)
What you should see in the terminal
Output
https://staging.myapp.com
chromium
5000
True
3
Now check the types
# Add these lines below your print statements print(type(base_url)) print(type(timeout)) print(type(headless))
Output
<class 'str'>
<class 'int'>
<class 'bool'>
Notice: timeout = 5000 has no quotes — it's a number. timeout = "5000" with quotes would be text. Python treats them completely differently. Numbers can do maths. Text cannot.

Example 2 — Test user credentials str
Test users are something you'll store in every test project. Username, password, role, expected values after login — all strings.
Type this — it's a new section in day1.py
# Test user credentials username = "tomsmith" password = "SuperSecretPassword!" user_role = "admin" expected_message = "You logged into a secure area!" print(username) print(password) print(user_role) print(expected_message)
Output
tomsmith
SuperSecretPassword!
admin
You logged into a secure area!
Now update a variable and reuse it
# Change the value — just reassign it username = "wronguser" print(username) # now prints: wronguser # The old value is gone — Python only remembers the latest
Strings must always be in quotes. If you write username = tomsmith without quotes, Python thinks tomsmith is another variable — and will crash with NameError because that variable doesn't exist.

Example 3 — HTTP status codes int
Integers are whole numbers — no decimal point, no quotes. Status codes, timeout values, retry counts, port numbers — all integers in automation.
Type this
# HTTP status codes you'll assert in API tests status_ok = 200 status_created = 201 status_no_content = 204 status_not_found = 404 status_server_err = 500 print(status_ok) print(status_not_found) # Integers can do maths timeout_seconds = 5000 timeout_minutes = timeout_seconds / 60000 print(timeout_minutes) # 0.08333... # Compare integers actual_status = 200 print(actual_status == status_ok) # True print(actual_status == status_not_found) # False
Output
200
404
0.08333333333333333
True
False
The == operator compares two values and gives back True or False. This is the foundation of every assertion you'll ever write in test automation. When you later write assert response.status_code == 200 in an API test — that == is doing exactly this.

Example 4 — Test flags and switches bool
Booleans are only ever True or False — capital first letter, no quotes. Used for: headless mode on/off, test enabled/disabled, login state, feature flags.
Type this
# Boolean flags in test configuration headless_mode = True slow_motion = False take_screenshot = True record_video = False is_logged_in = False print(headless_mode) print(is_logged_in) print(type(headless_mode)) # <class 'bool'> # Booleans come from comparisons too status = 200 test_passed = (status == 200) # True test_failed = (status == 404) # False print(test_passed) print(test_failed) # not flips a boolean print(not test_passed) # False print(not is_logged_in) # True (not False = True)
Output
True
False
<class 'bool'>
True
False
False
True
Common mistake: writing headless_mode = true (lowercase t). Python will crash with NameError — it thinks true is a variable name. It must always be True and False with capital letters.

Example 5 — print() with labels putting it all together
Printing a bare value isn't useful in real tests. You need to print labels with values so you understand the output. Three ways to do it — you'll use all three this week.
Type this — all 3 print styles
# Method 1: comma-separated (simplest) url = "https://staging.myapp.com" status = 200 passed = True print("URL:", url) print("Status:", status) print("Passed:", passed) # Method 2: f-string (most useful — you'll use this every day) print(f"Testing URL: {url}") print(f"Response status: {status}") print(f"Test passed: {passed}") # Method 3: string concatenation with + (works but messy) # Note: + only works between strings # print("Status: " + status) ← this CRASHES print("Status: " + str(status)) # must convert int to str
Output
URL: https://staging.myapp.com
Status: 200
Passed: True
Testing URL: https://staging.myapp.com
Response status: 200
Test passed: True
Status: 200
Use f-strings from now on. The syntax is: put f before the quote, then wrap any variable in {curly braces}. It works with strings, integers, and booleans — no conversion needed.
