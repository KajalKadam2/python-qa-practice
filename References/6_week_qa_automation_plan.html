Phase 2
Weeks 3–4 — API testing + Selenium basics
Goal: Test REST APIs with Python + be able to talk about Selenium in interviews (still in 60% of job listings)
Market data: Playwright roles pay 15–24% MORE than Selenium roles, but Selenium is still in 56% of job postings. You need to know both — master Playwright, be familiar with Selenium.

Week 3 — Mon–Tue
API testing fundamentals: what is REST, HTTP methods (GET/POST/PUT/DELETE), status codes, headers, JSON. Postman: make your first API calls to reqres.in (free practice API)
Week 3 — Wed–Thu
Python requests library: write API tests with GET, POST, assert status codes and JSON response fields. Practice on reqres.in and jsonplaceholder.typicode.com
Week 3 — Fri + Weekend
Build Portfolio Project #2: API test suite. 10+ tests covering GET list, GET single, POST create, PUT update, DELETE. Push to a new GitHub repo "api-testing-python"
Week 4 — Mon–Tue
Selenium WebDriver basics: setup, finding elements, clicks, waits. Write the same login test you wrote in Playwright — now in Selenium. Feel the difference. Understand why Playwright is better.
Week 4 — Wed–Thu
Playwright advanced: network interception (mock API responses), file uploads, multiple tabs, iframes. These appear in senior interview questions.
Week 4 — Fri + Weekend
SQL basics: SELECT, WHERE, JOIN, COUNT. Practice on sqlzoo.net or leetcode easy SQL. Interviewers ask: "how would you verify data in the DB after an API call?"
End of week 4 — you must have these
API test suite (Python requests + pytest) live on GitHub
Can explain REST methods, status codes, and JSON in an interview
Written at least one Selenium test — can compare Selenium vs Playwright
Basic SQL SELECT queries — can query a DB to verify test data

Phase 4
Week 6 — Interview sprint + keep applying
Goal: Answer every common automation interview question confidently. Keep sending 5+ applications per day.
Your 7.4 years is your biggest asset in interviews. Every answer should connect your manual QA experience to the automation concept. "I used to do this manually, now I automate it because..."
Mon — Framework questions
Practice answering: What is POM? Why Playwright over Selenium? How do fixtures work? What is parametrize? Explain your test folder structure. Answer out loud — not just in your head.
Tue — Coding round prep
32% of SDET interviews now include coding tests. Practice: write a function, debug a broken test script, write a class. Use HackerRank "Easy" level Python problems — 3 per day.
Wed — QA scenario questions
These are YOUR strength. "How would you test a login page?" "What's your test strategy for a new feature?" "How do you handle flaky tests?" "What's in your regression suite?" Answer using your 7.4yr experience + automation angle.
Thu — CI/CD + API questions
Practice: What is CI/CD? What does your pipeline do? What's the difference between GET and POST? What do you assert in an API test? How do you test an API without a UI? Walk through your GitHub Actions setup.
Fri — Mock interview
Do a full 45-min mock interview — ask me and I'll play interviewer. OR record yourself answering 10 questions on video and watch it back. Brutal but effective.
Weekend
Keep applying (5/day). Research each company before applying. Tailor 2 lines of your resume cover to their stack. Follow up on week 5 applications. Rest matters — you need to be sharp for calls.
