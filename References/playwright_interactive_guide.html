Interactive Playwright reference guide covering actions, locators, assertions, and waits

What you can do on a page
# Navigate to a URL page.goto("https://example.com") # Click a button or link page.click("button[type='submit']") # Type into an input field page.fill("#email", "test@example.com") # Press a keyboard key page.press("#search", "Enter") # Select dropdown option page.select_option("#country", "India") # Check a checkbox page.check("#agree-terms") # Hover over an element page.hover(".menu-item") # Take a screenshot page.screenshot(path="screenshot.png")
As a manual QA, you already know these actions — you do them manually every day. Playwright just does them in code.
How to find elements on the page
# 1. By role — BEST PRACTICE (accessibility-based) page.get_by_role("button", name="Submit") page.get_by_role("textbox", name="Email") page.get_by_role("link", name="Sign in") # 2. By text content — great for headings, labels page.get_by_text("Welcome back") # 3. By placeholder text — good for inputs page.get_by_placeholder("Enter your email") # 4. By label — links input to its label page.get_by_label("Password") # 5. By test ID — most stable for automation page.get_by_test_id("submit-btn") # data-testid="submit-btn" # 6. CSS selector — fallback when above won't work page.locator("#login-form .submit-button") # 7. XPath — last resort, avoid if possible page.locator("//button[@type='submit']")
Priority order to use
Use first
get_by_role, get_by_label, get_by_placeholder
Stable & readable
Use second
get_by_test_id (needs dev to add data-testid)
Most stable
Use third
CSS selectors with IDs or unique classes
OK fallback
Avoid
XPath, positional CSS like nth-child(3)
Brittle
Your manual QA instinct applies here too — a locator based on visible text ("Submit") is better than a deep CSS path. If the text changes, the test should fail. If only the structure changes, the test shouldn't.
Verifying things are correct — this is where your QA brain shines
from playwright.sync_api import Page, expect # Page-level assertions expect(page).to_have_title("Dashboard") expect(page).to_have_url("https://app.com/dashboard") # Element visibility expect(page.get_by_text("Welcome!")).to_be_visible() expect(page.get_by_role("button", name="Delete")).to_be_hidden() # Element state expect(page.get_by_role("button", name="Submit")).to_be_enabled() expect(page.get_by_role("button", name="Save")).to_be_disabled() expect(page.get_by_role("checkbox")).to_be_checked() # Text content expect(page.locator(".error-msg")).to_have_text("Invalid email") expect(page.locator(".welcome")).to_contain_text("Hello") # Input value expect(page.get_by_label("Email")).to_have_value("user@test.com") # Count of elements expect(page.locator(".product-card")).to_have_count(12) # Negate any assertion with NOT expect(page.locator(".spinner")).not_to_be_visible()
All Playwright assertions have auto-wait built in. They retry for up to 5 seconds before failing. You don't need explicit sleep() calls!
Handling timing and dynamic content
# Auto-wait — Playwright does this automatically # Every action waits for element to be ready page.click("#submit") # waits until clickable # Wait for a specific element to appear page.wait_for_selector(".success-message") # Wait for navigation after click with page.expect_navigation(): page.click("#login-btn") # Wait for a network request to finish with page.expect_response("**/api/users"): page.click("#load-users") # Wait for page to reach a load state page.wait_for_load_state("networkidle") # Only use sleep as last resort! import time time.sleep(2) # BAD — makes tests slow and fragile
Never use time.sleep() in Playwright tests. It makes tests slow and still flaky. Use expect() assertions instead — they wait smartly until the condition is true.
A complete, well-structured test file
import pytest from playwright.sync_api import Page, expect # ---- Setup: runs before EACH test ---- @pytest.fixture(autouse=True) def go_to_login(page: Page): page.goto("https://the-internet.herokuapp.com/login") # ---- Test 1: happy path ---- def test_valid_login(page: Page): page.get_by_label("Username").fill("tomsmith") page.get_by_label("Password").fill("SuperSecretPassword!") page.get_by_role("button", name="Login").click() expect(page.get_by_text("You logged into a secure area!")).to_be_visible() # ---- Test 2: negative case ---- def test_invalid_login(page: Page): page.get_by_label("Username").fill("wronguser") page.get_by_label("Password").fill("wrongpass") page.get_by_role("button", name="Login").click() expect(page.locator("#flash.error")).to_contain_text("Your username is invalid!") # ---- Test 3: parametrize = same test, multiple data ---- @pytest.mark.parametrize("username,password", [ ("", "pass"), ("user", ""), ("", ""), ]) def test_empty_fields(page: Page, username, password): page.get_by_label("Username").fill(username) page.get_by_label("Password").fill(password) page.get_by_role("button", name="Login").click() expect(page.locator("#flash")).to_be_visible()
The site https://the-internet.herokuapp.com is a free practice site made exactly for automation practice. Use it for all your exercises.
Locators
page.get_by_role("button", name="Submit") page.get_by_label("Email") page.get_by_placeholder("Search...") page.get_by_text("Welcome") page.get_by_test_id("submit-btn") page.locator("#id") .locator(".class") .locator("tag")
Actions
page.goto(url) page.click(selector) page.fill(sel, text) page.press(sel, "Enter") page.check(sel) page.hover(sel) page.select_option(sel, value) page.screenshot(path="shot.png")
Assertions (all auto-wait)
expect(page).to_have_title("...") expect(page).to_have_url("...") expect(locator).to_be_visible() expect(locator).to_be_hidden() expect(locator).to_be_enabled() expect(locator).to_be_disabled() expect(locator).to_have_text("...") expect(locator).to_contain_text("...") expect(locator).to_have_value("...") expect(locator).to_have_count(n) expect(locator).not_to_be_visible()
Useful run commands
pytest test_login.py # run file pytest test_login.py --headed # show browser pytest test_login.py -v # verbose output pytest test_login.py -k "valid" # run matching tests pytest --browser firefox # use Firefox pytest --slowmo 500 # slow down 500ms
