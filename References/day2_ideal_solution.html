# ───────────────────────────────────────── # Day 2 Challenge — Ideal Solution # day2_challenge.py # ───────────────────────────────────────── # ── Starting data ────────────────────────── env = "staging" username = "tomsmith" current_url = " https://staging.myapp.com/secure " page_title = " Secure Area " flash_message = " \n You logged into a secure area! \n " response_time_ms = 843 # ── Step 1: build login_url with f-string ── login_url = f"https://{env}.myapp.com/login" print(f"Login URL : {login_url}") # ── Step 2: clean current_url ────────────── clean_url = current_url.strip() print(f"Current URL : {clean_url}") # ── Step 3: clean page_title ─────────────── clean_title = page_title.strip() print(f"Page title : {clean_title}") # ── Step 4: flash message contains "secure area" ── flash_clean = flash_message.strip().lower() flash_ok = "secure area" in flash_clean print(f"Flash OK : {flash_ok}") # ── Step 5: URL starts with https ────────── url_secure = clean_url.startswith("https") print(f"URL secure : {url_secure}") # ── Step 6: URL ends with /secure ────────── on_secure_page = clean_url.endswith("/secure") print(f"On /secure : {on_secure_page}") # ── Step 7: response time under 2000ms ───── fast_enough = response_time_ms < 2000 print(f"Fast enough : {fast_enough}") # ── Step 8: overall result + summary ─────── all_passed = url_secure and on_secure_page and flash_ok and fast_enough overall = "PASS" if all_passed else "FAIL" print(f"{'=' * 25}") print(f"=== Login Test Result ===") print(f"{'=' * 25}") print(f"Environment : {env}") print(f"User : {username}") print(f"URL secure : {url_secure}") print(f"On /secure : {on_secure_page}") print(f"Flash OK : {flash_ok}") print(f"Fast enough : {fast_enough}") print(f"{'─' * 25}") print(f"Overall : {overall}")

Every important line explained
f"https://{env}.myapp.com/login"
Step 1 — f-string URL builder. The f before the quote activates f-string mode. {env} is replaced with the value of the variable env. Result: https://staging.myapp.com/login. If you change env = "prod", you get the prod URL automatically — no other changes needed.
clean_url = current_url.strip()
Step 2 — strip before using. The raw value has spaces on both ends. If you used current_url directly in startswith() or endswith(), those spaces would cause it to return False even though the URL is correct. Always strip first, store in a new variable, then use the clean version everywhere else.
flash_clean = flash_message.strip().lower() flash_ok = "secure area" in flash_clean
Step 4 — the most important pattern in UI testing. Two methods chained: strip removes outer whitespace and newlines, lower makes comparison case-insensitive. Then in checks if the substring exists anywhere in the string. This exact pattern is what you'll write in Playwright: "secure area" in page.locator("#flash").inner_text().strip().lower().
url_secure = clean_url.startswith("https")
Step 5 — using clean_url, not current_url. This is why we stored the stripped version in step 2. If you wrote current_url.startswith("https"), the leading spaces would mean it starts with a space, not "https", and you'd get False. Always work with the clean variable.
all_passed = url_secure and on_secure_page and flash_ok and fast_enough overall = "PASS" if all_passed else "FAIL"
Step 8 — combining booleans and inline condition. and returns True only if ALL conditions are True — one failure makes the whole thing False. The inline "PASS" if all_passed else "FAIL" picks a string based on the boolean. This is called a ternary expression — you'll see it constantly in test reporting code.
print(f"{'=' * 25}")
Separator line trick. "=" * 25 repeats the = character 25 times. Doing this inside an f-string {'=' * 25} lets you print a separator without hardcoding 25 equals signs. Change the number to change the width. Useful for any formatted test output.
The 6 mistakes most people made on this challenge
Mistake 1 — using current_url directly without stripping
# Wrong — leading space breaks startswith() url_secure = current_url.startswith("https") # False! # Correct — strip first, then check clean_url = current_url.strip() url_secure = clean_url.startswith("https") # True
The raw string starts with spaces, not "https". startswith() is exact — a space at position 0 makes it return False. This is the most common real-world bug in UI test automation too. Scraped text always needs cleaning.
Mistake 2 — not chaining .lower() on flash message
# Wrong — case-sensitive, fragile flash_ok = "secure area" in flash_message.strip() # Correct — lowercase both sides flash_ok = "secure area" in flash_message.strip().lower()
If the page ever changes "Secure Area" to "SECURE AREA" or "secure Area", your test breaks. Lowercasing both sides makes it case-insensitive and resilient to UI copy changes.
Mistake 3 — hardcoding the URL instead of using f-string
# Wrong — hardcoded, doesn't use env variable login_url = "https://staging.myapp.com/login" # Correct — built from env variable login_url = f"https://{env}.myapp.com/login"
Both produce the same output right now, but the f-string version automatically changes when env changes. Hardcoded URLs are a maintenance problem — you'd have to update every URL manually when switching environments.
Mistake 4 — checking response time with a string comparison
# Wrong — comparing int to string fast_enough = response_time_ms < "2000" # TypeError crash # Correct — compare int to int (no quotes) fast_enough = response_time_ms < 2000
2000 without quotes is an integer. "2000" with quotes is a string. You cannot compare a number to text with < — Python will crash with TypeError. Numbers never get quotes.
Mistake 5 — repeating strip() instead of reusing clean variable
# Repetitive — stripping the same variable 3 times url_secure = current_url.strip().startswith("https") on_secure_page = current_url.strip().endswith("/secure") print(current_url.strip()) # Better — strip once, reuse everywhere clean_url = current_url.strip() url_secure = clean_url.startswith("https") on_secure_page = clean_url.endswith("/secure") print(clean_url)
Both work, but stripping once and storing it is cleaner and faster. If you need to change how you clean the value, you only change one line. This is the DRY principle — Don't Repeat Yourself — fundamental to writing maintainable test code.
Mistake 6 — printing result without a label
# Hard to read in terminal output print(url_secure) # prints: True ← True what? print(flash_ok) # prints: True ← True what? # Always label your output print(f"URL secure : {url_secure}") print(f"Flash OK : {flash_ok}")
When a test runs and prints 5 lines of "True", you have no idea which check passed. Always print a label with the value. This habit directly translates to readable test output in pytest — it tells you exactly what succeeded or failed.
