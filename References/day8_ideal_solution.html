Function 1 log_result — *args and **kwargs together
detail_str = ", ".join(details) if details else "no details" meta_str = " ".join(f"{k}={v}" for k, v in meta.items())
Joining args and kwargs into strings cleanly. ", ".join(details) converts the tuple into a comma-separated string. f"{k}={v}" formats each kwarg as browser=chromium. Both use generator expressions — no manual loops needed. The ternary if details else "no details" handles the case where nothing is passed.
if meta_str: line += f" | {meta_str}"
Only append metadata section if there is any. If no kwargs were passed, meta_str is an empty string — falsy. The if meta_str guard means the output line stays clean without a trailing |.
Function 2 build_report — *results unpacked at the call site
print(build_report("Smoke Test Suite", *test_results, show_passed=False))
*test_results unpacks the list into individual positional args. Inside the function, results is a tuple of dicts. This is the key pattern — you build a list dynamically then unpack it with * at the call site. Without the *, results would be a tuple containing one item: the whole list. That would break len(results) and all the comprehensions.
failures[:max_failures]
Slicing to respect the max_failures limit. If there are 50 failures but max_failures=10, only the first 10 are shown. This is a keyword-only argument — it cannot be passed positionally because *results comes before it in the signature.
Function 3 assert_response — None safe default + tuple return
required_fields = required_fields or []
Safe default pattern for mutable arguments. The parameter default is None — not []. The first line inside the function converts None to an empty list. or [] is shorthand for if required_fields is None: required_fields = []. Both work — or [] is more concise.
elif body[field] is None: errors.append(f"Field '{field}' is None")
Check for None values, not just missing keys. A field can exist in the response body but be set to None — which is still invalid for most required fields. Checking existence alone misses this case. The elif means: only check the value if the key exists — preventing a KeyError.
Function 4 retry — the higher-order function
def retry(func: Callable, *args: Any, max_attempts: int = 3, **kwargs: Any):
max_attempts is keyword-only because *args comes before it. Any parameter after *args can only be passed by name — never positionally. This means retry(my_func, "url", 5) would pass 5 as another positional arg, not as max_attempts. You must write retry(my_func, "url", max_attempts=5). This is intentional — it prevents the caller from accidentally passing the wrong thing.
result = func(*args, **kwargs) if result: return result
This is what a higher-order function does. func is just a variable that holds a function. func(*args, **kwargs) calls whatever function was passed in, with whatever args and kwargs were passed in. The retry wrapper does not know or care what func does — it just calls it and checks if it returned something truthy. This is how pytest's retry plugin, Playwright's wait helpers, and every decorator in Python works internally.
except Exception as e: print(f" ✗ Attempt {attempt}/{max_attempts} — error: {e}")
Catch exceptions so the retry continues. Without the try/except, a crash inside func would bubble up and stop all retries. Wrapping in try/except means an exception on attempt 1 is logged and retried on attempt 2. The function only gives up after all attempts are exhausted.
The 5 mistakes most people made
Mistake 1 — not joining *args into a string for log_result
# Wrong — prints the raw tuple print(f"[{status}] {test_id} | Details: {details}") # Output: [PASS] TC02 | Details: ('login ok', 'element found') # Correct — join into a readable string detail_str = ", ".join(details) if details else "no details" print(f"[{status}] {test_id} | Details: {detail_str}") # Output: [PASS] TC02 | Details: login ok, element found
*args arrives as a tuple. Printing it directly shows the tuple brackets and quotes. ", ".join(details) produces clean human-readable output. Always join *args before displaying.
Mistake 2 — passing test_results without * to build_report
# Wrong — passes the list as ONE argument build_report("Suite", test_results) # Inside: results = ([{...}, {...}, {...}],) ← tuple of one list # len(results) = 1, not 4! # Correct — unpack the list into individual args build_report("Suite", *test_results) # Inside: results = ({...}, {...}, {...}, {...}) ← tuple of 4 dicts # len(results) = 4
The * at the call site is as important as the * in the definition. Without it, the whole list is one argument. With it, each item is a separate argument. This is the most common *args mistake.
Mistake 3 — using [] as default for required_fields
# Wrong — shared list across all calls (mutable default trap) def assert_response(status, body, expected_status=200, required_fields=[]): required_fields.append("something") # mutates the shared default! # Correct — None default, create list inside def assert_response(status, body, expected_status=200, required_fields=None): required_fields = required_fields or []
The mutable default trap from the lesson. If the function ever modifies the list (even accidentally), future calls share the contaminated list. Always use None and create the list inside.
Mistake 4 — retry crashes on exception instead of continuing
# Wrong — exception stops all retries while attempt < max_attempts: attempt += 1 result = func(*args, **kwargs) # crashes here — no more retries if result: return result # Correct — catch exceptions and continue retrying while attempt < max_attempts: attempt += 1 try: result = func(*args, **kwargs) if result: return result except Exception as e: print(f" ✗ error: {e}")
A retry wrapper that doesn't catch exceptions is useless. The whole point of retry is to handle transient failures — which includes crashes. Wrapping the call in try/except means any failure type (return value or exception) triggers a retry.
Mistake 5 — missing type hints or incomplete signature
# Incomplete — no type hints, no docstring def retry(func, *args, max_attempts=3, **kwargs): pass # Complete — typed, documented def retry( func: Callable, *args: Any, max_attempts: int = 3, **kwargs: Any ) -> Optional[Any]: """Call func(*args, **kwargs) up to max_attempts times."""
Type hints on *args and **kwargs use the item type, not the container. *args: Any means each individual arg is Any — not that args is Any. Callable from typing tells VS Code and other tools that func is a callable. These make your code self-documenting and enable autocomplete.
