Day 16 — pytest fixtures and conftest.py
You wrote 3 tests on Day 15 and each one had page.goto() at the top. Now imagine 50 tests — all repeating the same setup. Fixtures solve this — they run setup code automatically before your test, and cleanup code automatically after.

Today you learn the full fixture system — the backbone of every professional Playwright test suite.
The problem fixtures solve
# WITHOUT fixtures — copy-paste setup in every test def test_login(page): page.goto("https://the-internet.herokuapp.com/login") # repeated # ... test code def test_wrong_password(page): page.goto("https://the-internet.herokuapp.com/login") # repeated # ... test code def test_empty_fields(page): page.goto("https://the-internet.herokuapp.com/login") # repeated # ... test code # WITH fixtures — setup runs automatically def test_login(login_page): # fixture handles goto() # ... test code only def test_wrong_password(login_page): # fixture handles goto() # ... test code only
How fixtures work — the execution flow
pytest finds test
→
runs fixture setup
→
runs test function
→
runs fixture teardown
A fixture is just a function decorated with @pytest.fixture. When a test function has a parameter with the same name as a fixture, pytest automatically calls the fixture and passes its return value to the test. No special import needed — pytest discovers them automatically.

Your first fixture — step by step
Write this in your test file first. Later you will move fixtures to conftest.py so all test files can share them.
Create test_fixtures.py and type this
import pytest from playwright.sync_api import Page, expect # ── Define a fixture ───────────────────────────── @pytest.fixture def login_page(page: Page): """Navigate to the login page before each test.""" page.goto("https://the-internet.herokuapp.com/login") return page # hand the page back to the test # ── Use the fixture ────────────────────────────── # Note: parameter name 'login_page' matches fixture name above def test_valid_login(login_page: Page): login_page.get_by_label("Username").fill("tomsmith") login_page.get_by_label("Password").fill("SuperSecretPassword!") login_page.get_by_role("button", name="Login").click() expect(login_page.get_by_text("You logged into a secure area!")).to_be_visible() def test_invalid_login(login_page: Page): login_page.get_by_label("Username").fill("wronguser") login_page.get_by_label("Password").fill("wrongpass") login_page.get_by_role("button", name="Login").click() expect(login_page.locator("#flash.error")).to_contain_text("Your username is invalid!") def test_page_title(login_page: Page): expect(login_page).to_have_title("The Internet")
Run it
pytest test_fixtures.py -v --headed
Expected output
PASSED test_fixtures.py::test_valid_login
PASSED test_fixtures.py::test_invalid_login
FAILED test_fixtures.py::test_page_title
test_page_title FAILS — and that is correct. The login page title is "The Internet" but after our fixture navigates to /login the title might differ. This shows you fixtures run before EACH test independently. Notice: no page.goto() in any test function — the fixture handles it.

yield — running code after the test teardown pattern
When you use yield instead of return in a fixture, everything before yield is setup and everything after yield is teardown. The teardown runs after the test finishes — whether the test passed or failed. This is how you guarantee cleanup.
The yield pattern
@pytest.fixture def login_page(page: Page): # ── SETUP — runs before the test ──────────── print("\n → Setting up: navigating to login") page.goto("https://the-internet.herokuapp.com/login") yield page # ← test runs HERE, receives 'page' # ── TEARDOWN — runs after the test ────────── print("\n → Teardown: test finished, browser closing")
Type this complete example — notice the yield
import pytest from playwright.sync_api import Page, expect @pytest.fixture def authenticated_page(page: Page): """Log in before the test, log out after.""" # SETUP — log in page.goto("https://the-internet.herokuapp.com/login") page.get_by_label("Username").fill("tomsmith") page.get_by_label("Password").fill("SuperSecretPassword!") page.get_by_role("button", name="Login").click() print("\n → Setup: logged in successfully") yield page # test runs here # TEARDOWN — click logout page.get_by_role("link", name="Logout").click() print("\n → Teardown: logged out") def test_secure_area_visible(authenticated_page: Page): expect(authenticated_page.get_by_text("Secure Area")).to_be_visible() def test_logout_link_visible(authenticated_page: Page): expect(authenticated_page.get_by_role("link", name="Logout")).to_be_visible()
pytest test_fixtures.py -v -s --headed # -s shows print output
The -s flag shows print statements — useful during development to see setup/teardown messages. Remove it in CI/CD for clean output.

conftest.py — sharing fixtures across all test files most important file
conftest.py is a special file that pytest finds automatically. Fixtures defined in it are available to every test file in the same folder and all subfolders — no import needed. This is where shared fixtures live in every professional project.
Project structure — create these files
qa-automation/ ├── conftest.py ← shared fixtures here ├── test_login.py ← uses fixtures from conftest ├── test_checkboxes.py ← uses fixtures from conftest └── test_dropdowns.py ← uses fixtures from conftest
Create conftest.py in your qa-automation folder
# conftest.py import pytest from playwright.sync_api import Page # ── Base URL config ────────────────────────────── BASE_URL = "https://the-internet.herokuapp.com" # ── Fixture: navigate to login page ───────────── @pytest.fixture def login_page(page: Page): """Navigate to login page before each test.""" page.goto(f"{BASE_URL}/login") yield page # ── Fixture: already logged in ─────────────────── @pytest.fixture def logged_in_page(page: Page): """Login and navigate to secure area.""" page.goto(f"{BASE_URL}/login") page.get_by_label("Username").fill("tomsmith") page.get_by_label("Password").fill("SuperSecretPassword!") page.get_by_role("button", name="Login").click() yield page # teardown: navigate away to reset state page.goto(f"{BASE_URL}") # ── Fixture: test user data ────────────────────── @pytest.fixture def valid_user(): """Return valid test user credentials.""" return { "username": "tomsmith", "password": "SuperSecretPassword!", "expected_msg": "You logged into a secure area!" } @pytest.fixture def invalid_user(): """Return invalid test user credentials.""" return { "username": "wronguser", "password": "wrongpass", "expected_msg": "Your username is invalid!" }
No import needed in test files — pytest finds conftest.py automatically in the same folder. The page fixture that Playwright provides is also available in conftest.py — you just use it as a parameter.

Using conftest fixtures in test files
Once fixtures are in conftest.py, any test file in the same folder can use them just by adding the fixture name as a parameter. No import, no setup — just ask for it by name.
Create test_login.py — uses fixtures from conftest.py
# test_login.py — NO imports from conftest needed from playwright.sync_api import Page, expect # login_page comes from conftest.py automatically def test_valid_login(login_page: Page, valid_user): login_page.get_by_label("Username").fill(valid_user["username"]) login_page.get_by_label("Password").fill(valid_user["password"]) login_page.get_by_role("button", name="Login").click() expect(login_page.get_by_text(valid_user["expected_msg"])).to_be_visible() def test_invalid_login(login_page: Page, invalid_user): login_page.get_by_label("Username").fill(invalid_user["username"]) login_page.get_by_label("Password").fill(invalid_user["password"]) login_page.get_by_role("button", name="Login").click() expect(login_page.locator("#flash.error")).to_contain_text( invalid_user["expected_msg"] ) def test_secure_area(logged_in_page: Page): # logged_in_page fixture already did the login expect(logged_in_page).to_have_url(f"https://the-internet.herokuapp.com/secure") expect(logged_in_page.get_by_text("Secure Area")).to_be_visible() expect(logged_in_page.get_by_role("link", name="Logout")).to_be_visible()
Run all tests
pytest test_login.py -v --headed
Expected output
PASSED test_login.py::test_valid_login
PASSED test_login.py::test_invalid_login
PASSED test_login.py::test_secure_area
A test can use multiple fixtures at once — just add both as parameters: def test_something(login_page, valid_user). pytest injects both automatically. The order of parameters does not matter.

Fixture scopes — how often a fixture runs controls speed
By default, fixtures run before every single test. Scopes let you control this — run expensive setup once per file or once per session. This is how you make large test suites fast.
scope="function"
Default. Runs before every test function.
Use for: page navigation, test data
scope="class"
Runs once per test class.
Use for: shared class-level state
scope="module"
Runs once per test file.
Use for: expensive per-file setup
scope="session"
Runs once for the entire test run.
Use for: auth tokens, DB connections
Type this — see the difference
import pytest # Runs before EVERY test — default @pytest.fixture(scope="function") def function_fixture(): print("\n function_fixture: setting up") yield print("\n function_fixture: tearing down") # Runs once per FILE — faster @pytest.fixture(scope="module") def module_fixture(): print("\n module_fixture: setting up ONCE") yield {"token": "abc123"} print("\n module_fixture: tearing down ONCE") # Runs once for ALL tests in the whole session @pytest.fixture(scope="session") def session_fixture(): print("\n session_fixture: setting up ONE TIME") yield {"base_url": "https://the-internet.herokuapp.com"} print("\n session_fixture: done") def test_a(function_fixture, module_fixture, session_fixture): print(f"\n test_a: token={module_fixture['token']}") def test_b(function_fixture, module_fixture, session_fixture): print(f"\n test_b: url={session_fixture['base_url']}")
pytest test_scopes.py -v -s
What you will see — notice what runs once vs twice
session_fixture: setting up ONE TIME ← once
module_fixture: setting up ONCE ← once
function_fixture: setting up ← per test
test_a: token=abc123
function_fixture: tearing down ← per test
function_fixture: setting up ← per test
test_b: url=https://the-internet...
function_fixture: tearing down ← per test
module_fixture: tearing down ONCE ← once
session_fixture: done ← once
For Playwright tests: use scope="function" (default) for page fixtures — each test gets a fresh browser state. Use scope="session" for config data like base_url and credentials that never change.

autouse — fixtures that run without being requested screenshots on failure
autouse=True makes a fixture run for every test automatically — no parameter needed in the test function. The most common use in QA: automatically take a screenshot when a test fails. Every test gets this for free without listing the fixture name.
Add this to conftest.py — auto screenshot on failure
import pytest from playwright.sync_api import Page from pathlib import Path # ── Auto screenshot on test failure ───────────── @pytest.fixture(autouse=True) def screenshot_on_failure(page: Page, request): """Automatically screenshot any failing test.""" yield # test runs here # After test — check if it failed if request.node.rep_call.failed: # Create screenshots folder if needed Path("screenshots").mkdir(exist_ok=True) # Safe filename from test name test_name = request.node.name.replace("[","_").replace("]","") path = f"screenshots/{test_name}.png" page.screenshot(path=path, full_page=True) print(f"\n 📸 Screenshot saved: {path}") # This hook makes request.node.rep_call available @pytest.hookimpl(tryfirst=True, hookwrapper=True) def pytest_runtest_makereport(item, call): outcome = yield rep = outcome.get_result() setattr(item, f"rep_{rep.when}", rep)
When a test fails you now see
FAILED test_login.py::test_something
📸 Screenshot saved: screenshots/test_something.png
This is a real professional pattern. Every QA automation team uses automatic screenshots on failure. Without this, debugging a failing CI/CD test is nearly impossible. With it, you open the screenshot and see exactly what was on screen when it failed.

# conftest.py — complete production version import pytest from playwright.sync_api import Page from pathlib import Path # ── Central config ─────────────────────────────── BASE_URL = "https://the-internet.herokuapp.com" VALID_USER = { "username": "tomsmith", "password": "SuperSecretPassword!", "success_msg": "You logged into a secure area!" } INVALID_USERS = [ ("wronguser", "SuperSecretPassword!", "Your username is invalid!"), ("tomsmith", "wrongpassword", "Your password is invalid!"), ("", "", "Your username is invalid!"), ] # ── Page fixtures ──────────────────────────────── @pytest.fixture def login_page(page: Page): page.goto(f"{BASE_URL}/login") yield page @pytest.fixture def logged_in_page(page: Page): page.goto(f"{BASE_URL}/login") page.get_by_label("Username").fill(VALID_USER["username"]) page.get_by_label("Password").fill(VALID_USER["password"]) page.get_by_role("button", name="Login").click() yield page # ── Data fixtures ──────────────────────────────── @pytest.fixture def valid_user(): return VALID_USER @pytest.fixture def invalid_users(): return INVALID_USERS # ── Auto screenshot on failure ─────────────────── @pytest.hookimpl(tryfirst=True, hookwrapper=True) def pytest_runtest_makereport(item, call): outcome = yield rep = outcome.get_result() setattr(item, f"rep_{rep.when}", rep) @pytest.fixture(autouse=True) def screenshot_on_failure(page: Page, request): yield if hasattr(request.node, "rep_call") and request.node.rep_call.failed: Path("screenshots").mkdir(exist_ok=True) name = request.node.name.replace("[","_").replace("]","") page.screenshot(path=f"screenshots/{name}.png", full_page=True) print(f"\n 📸 Screenshot: screenshots/{name}.png")
